# 阶段1：路径确认 + 需求澄清 + 现状理解

## Part A: 输出位置确认

依次询问以下信息（已提供则跳过）：

| 信息项 | 询问时机 | 示例 |
|--------|----------|------|
| 项目集 | 未提供时 | tms / erpp / wms |
| 系统 | 项目集确认后 | ETMS / GPS / RMS |
| 需求类型 | 系统确认后 | 主版本_功能优化 / 项目定制化功能 |
| 需求名称 | 类型确认后 | 费用审批优化 |

**输出路径构建：**
`{文档仓库根目录}/{项目集}/{系统}/需求文档/{需求类型}/{需求名称}/`

---

## Part B: 智能检测（继续分析模式）

**在进入Part C之前，执行智能检测：**

检测 `需求调研/` 目录是否存在核心点文档（`需求文档核心点_*.md`）：

| 检测结果 | 处理方式 |
|----------|----------|
| 无核心点文档 | 正常流程 → Part C（过程文件确认） |
| 存在核心点文档 | 询问用户选择模式 |

**询问方式：**

> "检测到已有核心点文档（当前版本：v{N}），请选择：
> 
> | 选项 | 说明 |
> |------|------|
> | **A. 继续分析** | 基于上次结果，结合新澄清内容继续分析 |
> | **B. 重新开始** | 从零开始，生成新的核心点文档 |
> 
> 请输入选择：A 或 B"

**用户选择处理：**

| 选择 | 处理 |
|------|------|
| A（继续分析） | 跳转到 Part F（继续分析模式） |
| B（重新开始） | 继续正常流程 → Part C |

---

## Part C: 过程文件确认

询问需求澄清的过程文件位置：
- 笔记文件、会议纪要、需求清单、录音转写
- 格式：.md 或 .txt

**文件命名建议：**
- 按日期命名：`2026-05-13_第一次澄清.md`
- 或按内容命名：`费用审批流程澄清.md`

---

## Part D: 现状材料确认（可选）

询问是否需要阅读现有系统材料：
- 现有前端代码/原型
- 系统架构文档
- 历史需求文档
- 按需提供，节省token

---

## Part E: 项目基础信息提取

**优先从过程文件提取：**
- 客户名称/编码
- 业务机会名称/编码
- 项目联系人

**询问补充：**
- 需求负责人（默认当前用户）
- 时间线期望（如有）

**记忆复用：**
从memory读取历史配置，询问"是否沿用上次信息？"

---

## Part F: 文件夹初始化

用户确认后创建目录结构：

```
{需求名称}/
├── 需求调研/
│   ├── 需求文档核心点_v1.md    ← 核心点文档（版本文件）
│   ├── 需求文档核心点_v2.md
│   └── 处理记录.json           ← 处理历史记录
├── 需求澄清/                   ← 澄清笔记存放目录
│   ├── 2026-05-13_第一次澄清.md
│   └── 2026-05-15_第二次澄清.md
├── 需求文档/
├── 技术文档/
├── 测试文档/
├── 验收文档/
├── 其它文档/
└── HTML原型/
```

---

## Part G: 继续分析模式

**当用户选择"继续分析"时执行：**

### Step 1: 读取处理记录

读取 `需求调研/处理记录.json`：

```json
{
  "需求名称": "费用审批优化",
  "创建时间": "2026-05-13",
  "当前版本": "v1",
  "处理记录": [
    {
      "序号": 1,
      "笔记文件": "2026-05-13_第一次澄清.md",
      "状态": "已使用",
      "阅读时间": "2026-05-13 14:00",
      "生成结果": "需求文档核心点_v1.md",
      "内容摘要": "确定了审批流程、权限配置"
    },
    {
      "序号": 2,
      "笔记文件": "2026-05-15_第二次澄清.md",
      "状态": "待使用",
      "阅读时间": null,
      "生成结果": null,
      "内容摘要": null
    }
  ]
}
```

### Step 2: 扫描需求澄清目录

对比处理记录与 `需求澄清/` 目录中的文件，识别新增笔记。

### Step 3: 确认新增笔记

> "检测到新增笔记：
> 
> - `{笔记文件名}`
> 
> 是否基于此笔记继续分析？"

| 用户回答 | 处理 |
|----------|------|
| 确认 | 继续分析 |
| 提供其他文件 | 更新待处理笔记列表 |

### Step 4: 进入阶段2（继续分析模式）

带着以下内容进入阶段2：
- 当前版本核心点文档
- 新增的澄清笔记
- 处理记录.json

---

展示信息摘要：

```markdown
## 信息确认

- 需求负责人：{负责人}

```
