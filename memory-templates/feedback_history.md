# 反馈历史

**Why:** 避免重复调校

**How to apply:** 每次输出遵循此偏好

## 格式偏好

### 章节编号

使用中文：一、二、三...

### 优先级标记

`P0` `P1` `P2`（代码块格式）

### 空章节

填写"无"，不删除结构

## 流程图风格

- `flowchart TD`（从上到下）
- 中文节点名
- 判断用菱形 `{}`

## 原型注释

`<!-- [新增] -->` `<!-- [修改] -->` `<!-- [删除] -->`

需注明需求章节

## 甘特图

独立章节，不替换原有时间线

使用 `mermaid gantt`

## 交互偏好

### 确认方式

- 简单修改：对话告诉AI
- 复杂修改：用户自编辑，回复"已确认"

### Git操作

Skill不自动执行，用户自主管理

## HTML原型阶段

**支持独立调用**

除了完整流程中执行，还可独立调用：
- `/prd-workflow html`
- "生成HTML原型" / "绘制原型"

**独立调用流程：**
1. 询问PRD文档位置
2. 询问生成方式（修改现有/新增页面）
3. 询问现有材料来源（组件配置索引/直接提供文件）
4. 读取分析并生成原型

**无论核心点判断结果如何，进入阶段时必须询问是否需要原型**

## PRD编写前最终澄清

**在开始编写PRD文档前，必须询问用户是否有补充信息**

询问方式：
> "在开始编写PRD文档之前，还有什么信息需要补充或澄清吗？请补充需要澄清的内容，或回答"无，直接输出PRD文档"开始编写。"

用户回答处理：
- 补充内容 → AI记录，再次询问
- "无，直接输出PRD文档" → 开始编写PRD
- "返回核心点" → 回退到阶段2
